# CampusMate 개인정보처리방침

시행일: 2026-02-24  
최종 수정일: 2026-02-24

CampusMate(이하 "앱")는 이용자의 개인정보를 중요하게 생각하며, 관련 법령을 준수하기 위해 노력합니다.  
본 방침은 앱에서 어떤 정보를 어떤 목적으로 처리하는지 설명합니다.

## 1. 개인정보처리자 및 연락처

- 서비스명: CampusMate
- 문의: minwooj02@gmail.com

## 2. 처리하는 정보

앱은 회원가입 없이 동작하며, 이용자가 입력한 데이터를 기기에 저장하는 로컬 중심 구조입니다.

1) 이용자가 직접 입력/저장하는 정보(기기 로컬 저장)
- 할 일, 강의명, PDF 메모, 시간표 이미지, 앱 설정값
- 학교 캘린더 연동을 위한 ICS URL

2) 기능 제공 과정에서 생성되는 기술 정보
- 로컬 알림 예약/설정 정보
- 백업 내보내기/복원 과정에서 생성되는 백업 파일

3) 제3자 SDK를 통해 처리될 수 있는 정보
- Google Mobile Ads SDK(AdMob): 광고 식별자, 기기/앱 진단 정보, 광고 제공/성과 측정 관련 정보

## 3. 처리 목적

- 할 일/캘린더/시간표/강의자료 기능 제공
- ICS 일정 조회 및 표시
- 로컬 알림 제공
- 백업/복원 기능 제공
- 광고 제공 및 서비스 안정성 개선

## 4. 저장 방식과 위치

- 핵심 사용자 데이터는 기본적으로 이용자 기기에 저장됩니다.
- ICS URL을 입력한 경우에만 해당 URL로 네트워크 요청이 발생합니다.
- 앱은 이용자 콘텐츠를 개발자 운영 서버로 별도 업로드하지 않습니다.

## 5. 보유 및 이용 기간

- 로컬 저장 데이터: 이용자가 삭제하거나 앱을 제거할 때까지
- 백업 파일: 이용자가 선택한 저장 위치에서 이용자가 직접 관리
- 제3자 SDK 처리 데이터: 각 사업자의 정책에 따름

## 6. 제3자 제공 및 처리위탁

앱은 이용자 콘텐츠를 판매하지 않으며, 계정 기반 사용자 DB 서버를 운영하지 않습니다.  
다만 아래 SDK 사업자가 각 정책에 따라 데이터를 처리할 수 있습니다.

1) Google Mobile Ads (AdMob)
- 목적: 광고 제공, 광고 성과 측정, 부정행위 방지
- 정책: https://policies.google.com/privacy
- 안내: https://support.google.com/admob/answer/6128543

## 7. 국외 이전 가능성

AdMob 사용 시, 사업자 인프라 운영 위치(해외 포함)로 데이터가 이전되어 처리될 수 있습니다.  
세부 사항은 해당 사업자 정책을 따릅니다.

## 8. 이용자 권리와 행사 방법

이용자는 아래 방법으로 개인정보 관련 권리를 행사할 수 있습니다.

- 앱 내 데이터 직접 삭제 또는 앱 삭제
- 백업 파일 직접 삭제
- 기기 OS 설정에서 광고 식별자 재설정/제한
- 문의: minwooj02@gmail.com

앱은 계정 기반 서버를 운영하지 않으므로, 개발자가 원격으로 개별 사용자 로컬 데이터를 직접 열람/삭제하는 기능은 제공하지 않습니다.

## 9. 안전성 확보 조치

- 백업 PIN 기반 백업 파일 암호화(선택)
- 입력/파일 경로 검증, 파일 크기 제한
- 비암호화(HTTP) ICS URL 차단

법령/기능/운영 정책 변경 시 본 방침이 업데이트될 수 있습니다.
