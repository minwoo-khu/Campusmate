# Google Play Data safety 작성 메모 (초안)

기준일: 2026-02-24

## 앱 개요

- 앱명: CampusMate
- 성격: 로컬 중심(Local-first) 학습/생산성 앱

## 수집/공유 기본 원칙

- 개발자 서버에 사용자 콘텐츠를 별도 저장하지 않음
- 사용자 입력 데이터는 기본적으로 기기 로컬 저장
- 네트워크 사용은 제한적
  - 사용자가 등록한 ICS URL 조회
  - 광고 SDK(AdMob) 동작

## 데이터 범주 (앱 기능 데이터)

- 할 일, 강의, PDF 메모, 시간표 이미지, 설정값
- ICS URL
- 알림 예약 정보
- 백업/복원 파일 (사용자 선택 경로 저장)

## 처리 목적

- 핵심 기능 제공(할 일/캘린더/시간표/강의자료)
- 로컬 알림 제공
- 백업/복원 기능 제공
- 광고 제공 및 앱 운영 안정화

## 광고 SDK 관련 체크 포인트

- Google Mobile Ads SDK(AdMob) 사용
- Play Console `App content > Ads`에서 `Contains ads` 설정
- Data safety에 광고 관련 데이터 처리 항목 반영
- 개인정보처리방침에 AdMob 관련 링크/설명 포함

## 보안 메모

- 백업 PIN 기반 암호화 지원
- 입력/파일 경로 검증 및 파일 크기 제한 적용
- ICS는 HTTPS URL만 허용

## 작성 시 주의

- 최종 제출 전, 실제 배포 빌드 SDK 구성과 체크 항목을 다시 대조
- 문서/스토어 설명/정책 문구 간 표현 불일치가 없도록 통일
