# Google Play Data Safety 작성 초안

최종 업데이트: 2026-02-12

## 앱 개요
- 앱명: CampusMate
- 성격: 로컬 우선(Local-first) 생산성 앱

## 수집/공유 여부(초안)
- 개발자 서버로 사용자 데이터를 수집/공유하지 않음
- 사용자가 입력한 데이터는 기본적으로 기기에 로컬 저장
- 사용자가 입력한 ICS URL이 있는 경우에만 해당 외부 캘린더 조회

## 데이터 항목(앱 기능 데이터)
- 할 일, 강의, PDF 메모, 시간표 이미지, 설정값

## 처리 목적
- 앱 핵심 기능 제공(일정 관리, 리마인더, 백업/복원)

## 전송/보안 관련 메모
- 광고 SDK/추적 SDK 없음
- 백업 PIN 사용 시 백업 파일 암호화 지원

## 크래시 리포팅(Sentry) 사용 시 검토 항목
- 기본값은 비활성화이며 `ENABLE_SENTRY=true` 및 `SENTRY_DSN` 설정 시에만 활성화
- 활성화 시 앱 안정성 분석 목적의 기술적 진단 정보(예: 크래시 스택트레이스, 기기/OS/앱 버전) 전송 가능
- 광고/마케팅 목적 사용 없음, 내부 품질 모니터링 용도로만 사용

## 주의
- 최종 제출 전 실제 배포 빌드 기준으로 Play Console 문항과 대조 필요
- 크래시 리포팅(Sentry) 활성화 시 전송되는 진단 데이터 범위 별도 검토
